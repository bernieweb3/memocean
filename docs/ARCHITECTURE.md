# MemOcean v1 - System Architecture

## 1. Project Overview
MemOcean is an AI coding memory and project-aware mentoring platform. It is the Web2 port of "Kage Sensei", replacing decentralized storage (Walrus/Arweave) with a high-performance Cloudflare stack (R2, D1, KV) while maintaining Zero-Knowledge E2EE and MCP Streamable HTTP support.

## 2. Technology Stack
- **Runtime:** Node.js 22+ / Bun, TypeScript (Strict mode).
- **Edge Compute:** Cloudflare Workers (Hono framework).
- **Storage:** Cloudflare R2 (S3-compatible, E2EE blobs), Cloudflare D1 (SQLite, FTS5 metadata), Cloudflare KV (Cache, Rate-limits).
- **Protocol:** MCP (Model Context Protocol) via Streamable HTTP and STDIO.
- **Security:** AES-256-GCM E2EE, HKDF key derivation. Server NEVER sees plaintext data.

## 3. Core Modules

### 3.1. `packages/core` (The Brain)
- **E2EE Engine:** AES-256-GCM encryption. Keys derived via HKDF (Master -> Project -> Session).
- **CCR (Contextual Compression Ratio):** Calculates relevance scores and applies Token Budgeting to fit memories into LLM context windows.

### 3.2. `packages/storage` (The Web2 Adapters)
- **R2Adapter:** Handles `put()`, `get()`, `delete()` for encrypted blobs.
- **D1Adapter:** Handles metadata, FTS5 search queries, session graphs, and `lessons` (rules generated by Learn V2).
- **KVAdapter:** Handles API key validation, rate-limiting, and ephemeral MCP session states.

### 3.3. `apps/mcp-server` (The Interface)
- Built with **Hono** and `@modelcontextprotocol/sdk`.
- Supports **Streamable HTTP** (`POST /mcp`, `GET /mcp`, `DELETE /mcp`) for remote AI agents.
- Supports **STDIO** for local CLI execution.
- **MCP Tools:**
  - `ocean_remember`: Save encrypted memory.
  - `ocean_recall`: FTS5 search via D1.
  - `ocean_context`: Fetch and budget memories for LLM context.
  - `ocean_analyze` / `ocean_learn`: Trigger Learn meta-learning on source code.

### 3.4. `packages/cli` (The Bridge)
- Local CLI tool to zip `src/` directories and upload them securely to the MemOcean API.
- Triggers server-side `Learn` to extract project rules/lessons.

## 4. Database Schema (Cloudflare D1)
- `projects`: id, name, owner_id, created_at.
- `sessions`: id, project_id, agent_type, parent_session_id (for Session Graph).
- `memories`: id, session_id, r2_key, relevance_score, token_count, summary (plaintext for FTS5).
- `lessons`: id, project_id, rule_type, title, content, confidence_score.
- **FTS5 Virtual Tables:** `memories_fts` and `lessons_fts` for lightning-fast search.

## 5. Strict Rules for AI Coding Agents
1. **NEVER** store plaintext source code or raw prompts in R2. Everything must be E2EE encrypted by the client/SDK before upload.
2. **ONLY** plaintext metadata (summaries, tags, FTS5 indexes) is stored in D1.
3. **DO NOT** use Walrus, Arweave, or Avalanche. This is a strict Web2 Cloudflare stack.
4. Use Hono for all HTTP routing to ensure compatibility with Cloudflare Workers.
5. All MCP endpoints must support the latest Streamable HTTP specification.
